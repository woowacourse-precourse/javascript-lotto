# 🎐 로또 게임 기능 구현

## ▶ 기능 목록

### 1️⃣ `class` App

1. 로또를 구입한다. - **`class` PurchaseLotto**
   - [ ] 로또 구입 개수만큼 생성된 로또 번호들을 변수화한다. - **`var` purchasedLottoNumbers**

<br>

2. 당첨 번호를 생성한다. - **`class` Lotto**
   - [ ] 당첨 번호 6개를 변수화한다. - **`var` winningNumber**
   - [ ] 보너스 번호 1개를 변수화한다. - **`var` bonusNumber**

<br>

3. 로또 번호와 당첨 번호를 비교해 당첨 내역을 출력한다. - **`class` ResultOfLotto**
   - [ ] 당첨 내역을 출력한다. - **ResultOfLotto.result()**
   - [ ] 수익률을 출력한다. - **ResultOfLotto.yield()**

<br>

4. 게임을 종료시킨다.

### 2️⃣ `class` PurchaseLotto

1. 로또 구입 금액을 입력 받는다. - **`method` purchaseLottoInput()** 
   - [ ] 로또 구입 금액을 입력 받는다. - **`var` purchasePrice**
   - [ ] 입력값의 validation을 통해 잘못된 입력값인 경우 예외를 발생시킨다. - **`method` purchasePriceValidate(price)** 
   - [ ] 입력값이 valid한 경우 구입 금액을 반환한다.

<br>

2. 구입 금액에 해당하는 만큼 로또를 발행한다. - **`method` howManyLotto(price)**
   - [ ] 구입 가능한 로또 개수를 출력한다. - **`var` lottoCount**
   - [ ] 구입 가능한 로또 개수를 반환한다.

<br>

3. 로또 구입 개수만큼 6개의 로또 번호를 랜덤으로 생성한다. - **`method` createLottoNumber(count)**
   - [ ] 로또 구입 개수만큼 로또 번호 랜덤 생성을 반복한다. - **`var` totalLottoNumbers**
   - [ ] 6개의 로또 번호를 랜덤 생성한다. - **`var` lottoNumber**
   - [ ] 로또 구입 개수만큼 생성된 로또 번호들을 반환한다.

<br>

4. 로또 구입 금액의 validation - **`method` purchasePriceValidate(price)**
   - [ ] 숫자가 아닌 문자(특수문자 포함)이면 throw
   - [ ] 1000원 이하의 금액이면 throw
   - [ ] 1000원으로 나누어 떨어지지 않으면 throw

### 3️⃣ `class` Lotto

1. 당첨 번호 6개를 입력 받는다. - **`method` winningNumberInput()**
   - [ ] 당첨 번호 6개를 입력 받는다. - **`var` winningNumber**
   - [ ] 입력값의 validation을 통해 잘못된 입력값인 경우 예외를 발생시킨다. - **`method` winningNumberValidate(number)** 
   - [ ] 입력값이 valid한 경우 당첨 번호를 반환한다.

<br>

2. 보너스 번호 1개를 입력 받는다. - **`method` bonusNumberInput()**
   - [ ] 보너스 번호 1개를 입력 받는다. - **`var` bonusNumber**
   - [ ] 입력값의 validation을 통해 잘못된 입력값인 경우 예외를 발생시킨다. - **`method` bonusNumberValidate(number)** 
   - [ ] 입력값이 valid한 경우 보너스 번호를 반환한다.

<br>

4. 당첨 번호의 validation - **`method` winningNumberValidate(number)**
   - [ ] 숫자가 아닌 문자가 있으면 throw
   - [ ] 번호가 6개가 아니면 throw
   - [ ] 숫자가 중복되면 throw
   - [ ] 번호가 1 ~ 45까지의 수가 아니면 throw

<br>

1. 보너스 번호의 validation - **`method` bonusNumberValidate(number)**
   - [ ] 숫자가 아닌 문자(특수문자 포함)가 있으면 throw
   - [ ] 번호가 1개가 아니면 throw
   - [ ] 보너스 번호가 당첨 번호 6개와 중복되면 throw
   - [ ] 번호가 1 ~ 45까지의 수가 아니면 throw


### 4️⃣ `class` ResultOfLotto

1. 생성자 - **constructor(purchasedNumbers, winningNumber, bonusNumber)**
   - [ ] 사용자가 구매한 로또 번호, 당첨 번호, 보너스 번호
   - [ ] 당첨 기준과 금액 - **`var` winningCriteria**(object)
   - [ ] 사용자가 구매한 로또 번호의 각 당첨 등수 - **`var` lottoResults**

<br>

2. 사용자가 구매한 로또 번호와 당첨 번호를 비교한다. - **`method` compareNumbers(purchasedNumber)**
   - [ ] 사용자가 구매한 로또 번호와 당첨 번호를 비교한다.
   - [ ] 일치하는 번호 개수에 따라 당첨 등수를 반환한다.

<br>

3. 당첨 내역을 출력한다. - **`method` result()**
   - [ ] 로또 구입 개수만큼 생성된 로또 번호들을 당첨 번호와 비교한다.
   - [ ] 사용자가 구매한 로또 번호의 각 당첨 등수를 기록한다. 
   - [ ] 당첨 내역을 출력한다.

<br>

4. 당첨 내역에 따라 사용자의 수익률을 출력한다. - **`method` yield()**
   - [ ] 수익률을 출력한다.
   - [ ] 수익률을 반환한다.

## ▶ 기능 요구 사항

1. 로또 구입 금액을 입력 받는다.  
   - ex) 8000
   - ❗ **예외 사항**
     - 숫자가 아닌 문자(특수문자 포함)을 입력받는 경우
     - 1000원 이하의 금액을 받는 경우
     - 1000원으로 나누어 떨어지지 않는 경우

<br>

2. 구입 금액에 해당하는 만큼 로또를 발행한다.
   - 로또 1장의 가격은 1,000원이다.
   - 결론) 로또 구입 금액은 1,000원 단위로 받으므로, 1,000으로 나눴을 때 몫에 해당하는 값만큼 로또를 구입할 수 있다.
   - ex) 구입 금액이 8000원인 경우  
    8개를 구매했습니다.

<br>

3. 로또 구입 개수만큼 6개의 로또 번호를 랜덤으로 생성한다.
   - 로또 번호의 숫자 범위는 1 ~ 45까지이며 중복되지 않는다.
   - 6개의 로또 번호는 리스트 형태로 반환한다.
   - ex) 로또 구입 개수 = 8개인 경우  
    [8, 21, 23, 41, 42, 43]  
    [3, 5, 11, 16, 32, 38]  
    [7, 11, 16, 35, 36, 44]  
    [1, 8, 11, 31, 41, 42]  
    [13, 14, 16, 38, 42, 45]  
    [7, 11, 30, 40, 42, 43]  
    [2, 13, 22, 32, 38, 45]  
    [1, 3, 5, 14, 22, 45]

<br>

4. 당첨 번호 6개를 입력 받는다.
   - 당첨 번호의 숫자 범위는 1 ~ 45까지이며 중복되지 않는다.
   - 당첨 번호는 쉼표를 기준으로 구분한다. 
   - ex) 1,2,3,4,5,6
   - ❗ **예외 사항**
     - 숫자가 아닌 문자가 있는 경우
     - 번호가 6개가 아닌 경우
     - 숫자가 중복되는 경우
     - 번호가 1 ~ 45까지의 수가 아닌 경우
 
<br>

1. 보너스 번호 1개를 입력 받는다.
   - 보너스 번호의 숫자 범위는 1 ~ 45까지이다.
   - 당첨 번호 6개와 중복되지 않아야 한다.
   - ex) 7
   - ❗ **예외 사항**
     - 숫자가 아닌 문자(특수문자 포함)가 있는 경우
     - 번호가 1개가 아닌 경우
     - 보너스 번호가 당첨 번호 6개와 중복되는 경우
     - 번호가 1 ~ 45까지의 수가 아닌 경우

<br>

6. 사용자가 구매한 로또 번호와 당첨 번호를 비교하여 당첨 내역을 출력한다.
   - 당첨은 1등부터 5등까지 있다.
   - 당첨 기준과 금액은 아래와 같다.
     - 1등: 6개 번호 일치 👉 2,000,000,000원
     - 2등: 5개 번호 + 보너스 번호 일치 👉 30,000,000원
     - 3등: 5개 번호 일치 👉 1,500,000원
     - 4등: 4개 번호 일치 👉 50,000원
     - 5등: 3개 번호 일치 👉 5,000원
   - ex) **당첨 내역**  
    당첨 통계  
    **---**  
    3개 일치 (5,000원) - 1개  
    4개 일치 (50,000원) - 0개  
    5개 일치 (1,500,000원) - 0개  
    5개 일치, 보너스 볼 일치 (30,000,000원) - 0개  
    6개 일치 (2,000,000,000원) - 0개

<br>

7. 당첨 내역에 따라 사용자의 수익률을 출력하고 게임을 종료한다.
   - 수익률은 소수점 둘째 자리에서 반올림한다.
   - ex) 총 수익률은 62.5%입니다.


> 예외 상항 시, 아래와 같이 에러 문구를 출력해야한다.

- 사용자가 잘못된 값을 입력할 경우 `throw`문을 사용해 예외를 발생시킨다.
- 에러 문구는 `[ERROR]`로 시작해야 한다.

  ```plaintext
  [ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.
  ```
